## Problem

고등학생 때 학원 사무실 인프라를 한 번 깐 적이 있었다. 그 때 배운 걸 더 깊이 파보고 싶었다. 클라우드 서비스에 매달 돈 내는 게 아니라, **남는 PC + 인터넷만으로 같은 기능을 만들 수 있나?** 가 출발점.

목표: 외부에서 모바일 앱으로 접근 가능한 개인 클라우드 운영.

## Approach

전부 단독 구축 — 학원 인프라 경험을 토대로 처음부터 끝까지.

### 인프라 구성

- **OS** — Ubuntu LTS (안정성 우선)
- **방화벽** — iptables 규칙 직접 작성 (필요한 포트만 열기)
- **포트포워딩** — 공유기 NAT 설정으로 외부 접근 경로 확보
- **HTTPS** — 인증서 발급 + 갱신 자동화

### 서비스 구성

- **ownCloud** — 파일 동기화 (Dropbox 대체)
- **관계형 DB** — MariaDB로 ownCloud 메타데이터 관리

### 운영

- Putty/SSH로 원격 관리
- 외부에서 모바일 앱(ownCloud 클라이언트)으로 접근

## Insight

이 프로젝트는 *시작*이었다. 시스템을 "사용자"가 아니라 "관리자" 시각에서 보는 게 어떤 건지 처음 느꼈다 — 어떤 포트가 열려 있는지, 어디에 어떤 권한이 박혀 있는지, 인증서가 언제 만료되는지를 직접 책임지는 경험.

이 작업이 이후 **시스템 프로그래밍 → 백엔드 → Multi-Agent AI** 로 넘어가는 길의 출발점이 됐다. 추상화 레벨이 다르긴 했지만 "내가 직접 만든 시스템이 어떻게 굴러가나"를 들여다보는 태도가 일관되게 이어졌다.

GitHub 저장소는 따로 없다. 인프라 자체가 산출물이라 코드로 남길 게 적었기 때문. 이 자체가 첫 인프라 프로젝트의 한계였고, 이후 프로젝트들은 모두 코드로 재현 가능한 형태로 남기는 원칙을 세웠다.
